# システム概要

## 目的

Docker管理Webアプリケーションは、Dockerコンテナの管理とAnsible Playbookを使用した自動インストールを簡単に行うためのWebインターフェースです。コマンドライン操作なしで、ブラウザから直感的にコンテナを管理できます。

## 主要機能

### 1. コンテナ管理
- **コンテナ一覧表示**: 全てのDockerコンテナの状態を確認
- **起動/停止/再起動**: ワンクリックでコンテナ操作
- **ログ表示**: リアルタイムでコンテナログを閲覧
- **ステータス監視**: コンテナの実行状態をリアルタイム表示

### 2. Playbookインストール
- **デフォルトPlaybook**: 事前定義されたコンテナ（Nginx, MySQL, Redis, PostgreSQL, MongoDB, Node.js Webapp）
- **カスタムPlaybook**: GitリポジトリまたはYAMLファイルからPlaybookをダウンロード
- **環境変数設定**: インストール時にDB接続情報等を動的に設定
- **インストール履歴**: 実行ログの表示と結果確認

### 3. マルチドメイン対応（設計中）
- **ドメイン管理**: 複数ドメインとコンテナの紐付け
- **リバースプロキシ設定**: Nginx設定の自動生成
- **SSL/TLS管理**: Let's Encrypt証明書の自動更新

### 4. ネットワーク管理
- **Dockerネットワーク自動作成**: コンテナ間通信の簡素化
- **ネットワーク分離**: アプリケーションごとの独立したネットワーク

## 技術スタック

### バックエンド
- **言語**: Go 1.23.0
- **Webフレームワーク**: Gin 1.11.0
- **テンプレートエンジン**: html/template
- **YAML解析**: goccy/go-yaml 1.18.0

### フロントエンド
- **CSSフレームワーク**: Tailwind CSS v4

### インフラストラクチャ

- **プロセス管理**: systemd

### 開発ツール
- **ビルドツール**: Make
- **Node.js**: Tailwind CSSビルド用
- **Git**: バージョン管理

## ユースケース

### 1. 新規コンテナのインストール
```
ユーザー → インストール画面 → Playbook選択 → 
環境変数設定 → インストール実行 → 完了確認
```

### 2. 既存コンテナの管理
```
ユーザー → コンテナ一覧 → コンテナ選択 → 
操作（起動/停止/ログ確認）
```

### 3. カスタムアプリケーションのデプロイ
```
ユーザー → インストール画面 → URLからダウンロード → 
Git URLまたはYAML URL入力 → 環境変数設定 → デプロイ
```

## 対象ユーザー

- **システム管理者**: サーバーのコンテナを一元管理
- **開発者**: 開発環境のコンテナを素早く構築
- **DevOpsエンジニア**: インフラのコード化と自動デプロイ

## システム要件

### サーバー側
- **OS**: Linux (Ubuntu 20.04+推奨)
- **Docker**: 20.10+
- **Ansible**: 2.9+
- **Go**: 1.23+ (ビルド時のみ)
- **メモリ**: 512MB以上
- **ディスク**: 1GB以上

### クライアント側
- **ブラウザ**: Chrome, Firefox, Safari, Edge (最新版)
- **JavaScript**: 有効化必須
- **画面解像度**: 1024x768以上推奨

## セキュリティ

- **認証**: 現バージョンは認証なし（内部ネットワーク用）
- **パスワード管理**: type=passwordフィールドはマスク表示
- **Ansibleログ**: 機密情報はno_logで非表示化
- **ネットワーク**: ファイアウォールで58080ポートを適切に制限

## 制限事項

- 現バージョンではマルチユーザー非対応
- 同時実行は1つのPlaybookのみ（順次実行）
- コンテナ作成/削除機能は未実装（Playbook経由のみ）
- Dockerボリューム管理機能は未実装
